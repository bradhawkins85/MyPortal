How permissions are verified

  1. The portal acquires an application token using the stored company credentials.
  2. It locates the enterprise app's service principal in the Microsoft Entra ID tenant.
  3. It retrieves all app role assignments for that service principal and compares them against the expected permissions for each enterprise app.
  4. Each permission is reported as Pass (granted) or Fail (missing).

Missing permissions can be granted by re-running the auto-provision flow on the Office 365 Configuration page, or by manually assigning the roles in Microsoft Entra ID.
