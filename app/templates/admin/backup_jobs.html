Optional, e.g. Veeam — File server, nightly.

When enabled, once a Pass status has been recorded for the current day, any subsequent Warn or Fail report from the API will be ignored — the day's status remains locked to Pass. Useful for backup jobs that run multiple times a day against unreliable storage (e.g. external drives or remote NAS devices): as long as one run succeeds, the day is considered good regardless of later failures.
