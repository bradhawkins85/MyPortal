Provisioning fails? Create your own app registration

Some Azure AD tenants block the default public client used for provisioning (AADSTS700016). Follow these steps to register your own public client app and set M365_PKCE_CLIENT_ID.

Step-by-step: Create an Azure AD public client app registration
  1. Sign in to the Azure portal with a Global Administrator or Application Administrator account in your partner tenant.
  2. In the left navigation, go to Azure Active Directory → App registrations and click New registration.
  3. Enter a display name (e.g. MyPortal PKCE Bootstrap). Under Supported account types, select Accounts in this organizational directory only.
  4. Under Redirect URI, choose Public client/native (mobile & desktop) from the platform dropdown and enter the following callback URL:
    {{ request.url_for('m365_callback') }}
  5. Click Register.
  6. On the newly created app's page, click Authentication in the left menu.
  7. Scroll to Advanced settings. Under Allow public client flows, set Enable the following mobile and desktop flows to Yes.
  8. Click Save.
  9. Go to Overview and copy the Application (client) ID (a UUID like xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx).
  10. Add the following line to your .env file and restart MyPortal:
    M365_PKCE_CLIENT_ID=<your-application-client-id>
  11. Return to this page and click Provision admin app.

This app registration is only needed once during the initial CSP admin provisioning. After provisioning completes, MyPortal stores its own dedicated credentials and the PKCE bootstrap client is no longer used.

How CSP / GDAP mapping works

  1. Click Sign in as CSP account and authenticate with your Microsoft CSP or Lighthouse partner credentials.
  2. The portal reads your managed customer list from Microsoft Graph (GET /v1.0/contracts).
  3. Use the dropdown beside each customer to link their tenant to a MyPortal company.
  4. Once mapped, click Provision M365 to automatically create a dedicated enterprise app registration in that tenant.

GDAP relationships must already be established in Microsoft 365 Lighthouse or Partner Center before customers appear in this list.

Note: Once a tenant has been provisioned, the mapping and provisioning controls are locked. To modify or re-configure the connection, an administrator must first remove the M365 credentials via the company's settings page before changes can be made here.
