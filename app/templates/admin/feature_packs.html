About feature packs

Each pack lives under app/features/<slug>/ and exposes a FeaturePack manifest. Reloading imports the pack's modules fresh from disk, atomically swaps its router into the application, and waits for in-flight requests to drain. Other packs and the rest of the app continue serving traffic throughout.

Reloads succeed even if your changes are still in editor buffers — they only take effect after the new code is on disk. A failed reload leaves the previous version mounted and surfaces the error here.

See the API reference for the underlying POST /api/features/{slug}/reload endpoint, or docs/feature_packs.md in the repository for the authoring guide.
