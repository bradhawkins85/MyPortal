## User Flow

### Creating a New Critical Activity

1. User navigates to `/bcp/bia`
2. Clicks "Add Critical Activity" button
3. Redirected to `/bcp/bia/new`
4. Fills in:
   - **Basic Information**: Name, description, priority, importance, supplier dependency, notes
   - **Impact Assessment**: RTO, financial impact, costs, staffing, product/service, reputation, fines, legal liability, comments
5. Clicks "Create Activity"
6. Redirected back to `/bcp/bia` with success message
7. New activity appears in the sorted list

### Editing an Existing Activity

1. User navigates to `/bcp/bia`
2. Clicks "Edit" button on a specific activity row
3. Redirected to `/bcp/bia/{id}/edit`
4. Form is pre-populated with existing data
5. User modifies fields as needed
6. Clicks "Update Activity"
7. Redirected back to `/bcp/bia` with success message
8. Updated activity appears in the list

### Sorting Activities

1. User navigates to `/bcp/bia`
2. Uses the "Sort by" dropdown to select:
   - **Importance (1=Most Important)**: Shows activities by importance rating
   - **Priority (High to Low)**: Shows High priority first, then Medium, then Low
   - **Activity Name**: Shows alphabetically by name
3. Table reloads with new sort order

### Exporting to CSV

1. User navigates to `/bcp/bia`
2. Clicks "Export CSV" button
3. Browser downloads `bia_summary_YYYYMMDD_HHMMSS.csv`
4. File contains all activities with all fields for reporting

### Deleting an Activity

1. User navigates to `/bcp/bia`
2. Clicks "Delete" (X) button on a specific activity row
3. Browser shows confirmation dialog: "Delete this critical activity and all its impact data?"
4. User confirms
5. Activity and its associated impact data are deleted (CASCADE)
6. Page reloads with success message
7. Activity is removed from the list

## Responsive Design Notes

- Tables are responsive and scroll horizontally on small screens
- Forms stack vertically on mobile devices
- Buttons are touch-friendly sizes
- Priority and importance badges maintain visibility at all sizes
